## 三、数据库设计

### 3.1 ER 关系

```
User(1) ──< InternshipProject(N)          [teacher_id]
User(1) ──< Application(N)               [student_id]
InternshipProject(1) ──< Application(N)  [project_id]
Application(1) ──< Journal(N)            [application_id]
Application(1) ──< Report(N)             [application_id]
Application(1) ── Grade(1)               [application_id, UNIQUE]
Journal/Report/Project ──< Attachment(N) [ref_type + ref_id 多态关联]
```

### 3.2 建表 SQL

```sql
-- ============================================================
-- 数据库创建
-- ============================================================
CREATE DATABASE IF NOT EXISTS internship_db
    DEFAULT CHARACTER SET utf8mb4
    DEFAULT COLLATE utf8mb4_general_ci;

USE internship_db;

-- ============================================================
-- 1. 用户表 (sys_user)
-- 三种角色共用一张表, status 字段实现管理员审批注册
-- ============================================================
CREATE TABLE sys_user (
    id            BIGINT       NOT NULL AUTO_INCREMENT  COMMENT '主键',
    username      VARCHAR(50)  NOT NULL                 COMMENT '登录账号',
    password      VARCHAR(100) NOT NULL                 COMMENT '密码(BCrypt)',
    real_name     VARCHAR(50)  NOT NULL                 COMMENT '真实姓名',
    role          VARCHAR(20)  NOT NULL                 COMMENT '角色: STUDENT / TEACHER / ADMIN',
    status        VARCHAR(20)  NOT NULL DEFAULT 'PENDING' COMMENT '账号状态: PENDING / ACTIVE / DISABLED',
    student_no    VARCHAR(30)  NULL                     COMMENT '学号(学生专属)',
    college       VARCHAR(100) NULL                     COMMENT '学院',
    major         VARCHAR(100) NULL                     COMMENT '专业',
    grade         VARCHAR(20)  NULL                     COMMENT '年级(如2022级)',
    class_name    VARCHAR(50)  NULL                     COMMENT '班级',
    phone         VARCHAR(20)  NULL                     COMMENT '手机号',
    email         VARCHAR(100) NULL                     COMMENT '邮箱',
    avatar        VARCHAR(255) NULL                     COMMENT '头像URL',
    create_time   DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    update_time   DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted       TINYINT      NOT NULL DEFAULT 0       COMMENT '逻辑删除: 0否 1是',
    PRIMARY KEY (id),
    UNIQUE KEY uk_username (username)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';

-- ============================================================
-- 2. 实习项目表 (intern_project)
-- 教师发布, 含报名条件筛选与项目生命周期管理
-- ============================================================
CREATE TABLE intern_project (
    id              BIGINT       NOT NULL AUTO_INCREMENT  COMMENT '主键',
    title           VARCHAR(200) NOT NULL                 COMMENT '项目标题',
    description     TEXT         NULL                     COMMENT '项目详细描述',
    teacher_id      BIGINT       NOT NULL                 COMMENT '发布教师ID',
    company         VARCHAR(200) NULL                     COMMENT '实习单位',
    location        VARCHAR(200) NULL                     COMMENT '实习地点',
    max_students    INT          NOT NULL DEFAULT 0       COMMENT '最大招收人数(0=不限)',
    college_limit   VARCHAR(100) NULL                     COMMENT '限定学院(NULL=不限)',
    grade_limit     VARCHAR(20)  NULL                     COMMENT '限定年级(NULL=不限)',
    start_date      DATE         NULL                     COMMENT '实习开始日期',
    end_date        DATE         NULL                     COMMENT '实习结束日期',
    enroll_deadline DATE         NULL                     COMMENT '报名截止日期',
    semester        VARCHAR(30)  NULL                     COMMENT '所属学期(如2025-2026-1)',
    credit_type     VARCHAR(50)  NULL                     COMMENT '学分类型(专业实践/毕业实习等)',
    status          VARCHAR(20)  NOT NULL DEFAULT 'OPEN'  COMMENT '状态: OPEN / CLOSED / ARCHIVED',
    create_time     DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    update_time     DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted         TINYINT      NOT NULL DEFAULT 0,
    PRIMARY KEY (id),
    KEY idx_teacher (teacher_id),
    KEY idx_status (status),
    CONSTRAINT fk_project_teacher FOREIGN KEY (teacher_id) REFERENCES sys_user(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='实习项目表';

-- ============================================================
-- 3. 实习申请表 (intern_application)
-- project_id + student_id 唯一约束防止重复申请
-- phase 独立于审批状态, 跟踪实习进度
-- ============================================================
CREATE TABLE intern_application (
    id              BIGINT      NOT NULL AUTO_INCREMENT  COMMENT '主键',
    project_id      BIGINT      NOT NULL                 COMMENT '关联项目ID',
    student_id      BIGINT      NOT NULL                 COMMENT '申请学生ID',
    apply_reason    TEXT        NULL                     COMMENT '申请理由',
    status          VARCHAR(20) NOT NULL DEFAULT 'PENDING' COMMENT '审批状态: PENDING / APPROVED / REJECTED',
    review_comment  VARCHAR(500) NULL                    COMMENT '审批意见',
    reviewed_by     BIGINT      NULL                     COMMENT '审批人ID',
    reviewed_at     DATETIME    NULL                     COMMENT '审批时间',
    phase           VARCHAR(20) NOT NULL DEFAULT 'ENROLLED' COMMENT '实习阶段: ENROLLED / IN_PROGRESS / COMPLETED',
    create_time     DATETIME    NOT NULL DEFAULT CURRENT_TIMESTAMP,
    update_time     DATETIME    NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted         TINYINT     NOT NULL DEFAULT 0,
    PRIMARY KEY (id),
    UNIQUE KEY uk_project_student (project_id, student_id),
    KEY idx_student (student_id),
    CONSTRAINT fk_app_project FOREIGN KEY (project_id) REFERENCES intern_project(id),
    CONSTRAINT fk_app_student FOREIGN KEY (student_id) REFERENCES sys_user(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='实习申请表';

-- ============================================================
-- 4. 实习日志表 (intern_journal)
-- journal_type 区分日报/周报/阶段小结
-- ============================================================
CREATE TABLE intern_journal (
    id              BIGINT       NOT NULL AUTO_INCREMENT  COMMENT '主键',
    application_id  BIGINT       NOT NULL                 COMMENT '关联申请ID',
    student_id      BIGINT       NOT NULL                 COMMENT '学生ID',
    title           VARCHAR(200) NOT NULL                 COMMENT '日志标题',
    content         TEXT         NOT NULL                 COMMENT '日志内容',
    journal_type    VARCHAR(20)  NOT NULL DEFAULT 'DAILY' COMMENT '类型: DAILY / WEEKLY / PHASE_SUMMARY',
    journal_date    DATE         NOT NULL                 COMMENT '日志日期',
    week_no         INT          NULL                     COMMENT '周次(周报时填写)',
    teacher_comment VARCHAR(1000) NULL                    COMMENT '教师批注',
    score           DECIMAL(5,2) NULL                     COMMENT '教师评分(百分制)',
    scored_at       DATETIME     NULL                     COMMENT '评分时间',
    create_time     DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    update_time     DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted         TINYINT      NOT NULL DEFAULT 0,
    PRIMARY KEY (id),
    KEY idx_app (application_id),
    KEY idx_student (student_id),
    CONSTRAINT fk_journal_app FOREIGN KEY (application_id) REFERENCES intern_application(id),
    CONSTRAINT fk_journal_student FOREIGN KEY (student_id) REFERENCES sys_user(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='实习日志表';

-- ============================================================
-- 5. 实习报告表 (intern_report)
-- 支持教师评阅、驳回重提交、AI初审标记
-- ============================================================
CREATE TABLE intern_report (
    id              BIGINT       NOT NULL AUTO_INCREMENT  COMMENT '主键',
    application_id  BIGINT       NOT NULL                 COMMENT '关联申请ID',
    student_id      BIGINT       NOT NULL                 COMMENT '学生ID',
    title           VARCHAR(200) NOT NULL                 COMMENT '报告标题',
    summary         TEXT         NULL                     COMMENT '报告摘要',
    file_url        VARCHAR(500) NOT NULL                 COMMENT '报告PDF文件路径',
    semester        VARCHAR(30)  NULL                     COMMENT '绑定学期',
    credit_type     VARCHAR(50)  NULL                     COMMENT '绑定学分类型',
    status          VARCHAR(20)  NOT NULL DEFAULT 'SUBMITTED' COMMENT '状态: SUBMITTED / APPROVED / REJECTED / EXCELLENT',
    teacher_comment VARCHAR(1000) NULL                    COMMENT '教师评语',
    teacher_score   DECIMAL(5,2) NULL                     COMMENT '教师评分',
    is_excellent    TINYINT      NOT NULL DEFAULT 0       COMMENT '是否推荐优秀报告',
    ai_flag         VARCHAR(50)  NULL                     COMMENT 'AI初审标记(NORMAL/TEMPLATE_SUSPECTED/TOO_SHORT/MANUAL_REVIEW)',
    reviewed_by     BIGINT       NULL                     COMMENT '评阅教师ID',
    reviewed_at     DATETIME     NULL                     COMMENT '评阅时间',
    create_time     DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    update_time     DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted         TINYINT      NOT NULL DEFAULT 0,
    PRIMARY KEY (id),
    KEY idx_app (application_id),
    KEY idx_student (student_id),
    CONSTRAINT fk_report_app FOREIGN KEY (application_id) REFERENCES intern_application(id),
    CONSTRAINT fk_report_student FOREIGN KEY (student_id) REFERENCES sys_user(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='实习报告表';

-- ============================================================
-- 6. 成绩评定表 (intern_grade)
-- 一个申请对应一条成绩, 分项评分 + 管理员终审
-- ============================================================
CREATE TABLE intern_grade (
    id              BIGINT       NOT NULL AUTO_INCREMENT  COMMENT '主键',
    application_id  BIGINT       NOT NULL                 COMMENT '关联申请ID(唯一)',
    student_id      BIGINT       NOT NULL                 COMMENT '学生ID',
    project_id      BIGINT       NOT NULL                 COMMENT '项目ID',
    journal_score   DECIMAL(5,2) NULL                     COMMENT '日志综合得分',
    report_score    DECIMAL(5,2) NULL                     COMMENT '报告得分',
    final_score     DECIMAL(5,2) NULL                     COMMENT '最终总评成绩',
    teacher_id      BIGINT       NULL                     COMMENT '评定教师ID',
    teacher_comment VARCHAR(500) NULL                     COMMENT '教师综合评语',
    admin_status    VARCHAR(20)  NOT NULL DEFAULT 'PENDING' COMMENT '管理员审核: PENDING / CONFIRMED / RETURNED',
    admin_comment   VARCHAR(500) NULL                     COMMENT '管理员意见',
    confirmed_by    BIGINT       NULL                     COMMENT '审核管理员ID',
    confirmed_at    DATETIME     NULL                     COMMENT '审核时间',
    create_time     DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    update_time     DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted         TINYINT      NOT NULL DEFAULT 0,
    PRIMARY KEY (id),
    UNIQUE KEY uk_application (application_id),
    KEY idx_student (student_id),
    KEY idx_project (project_id),
    CONSTRAINT fk_grade_app FOREIGN KEY (application_id) REFERENCES intern_application(id),
    CONSTRAINT fk_grade_student FOREIGN KEY (student_id) REFERENCES sys_user(id),
    CONSTRAINT fk_grade_project FOREIGN KEY (project_id) REFERENCES intern_project(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='成绩评定表';

-- ============================================================
-- 7. 附件表 (sys_attachment)
-- 多态关联: ref_type + ref_id 关联到任意业务实体
-- ============================================================
CREATE TABLE sys_attachment (
    id          BIGINT       NOT NULL AUTO_INCREMENT  COMMENT '主键',
    ref_type    VARCHAR(30)  NOT NULL                 COMMENT '关联类型: JOURNAL / REPORT / PROJECT',
    ref_id      BIGINT       NOT NULL                 COMMENT '关联记录ID',
    file_name   VARCHAR(255) NOT NULL                 COMMENT '原始文件名',
    file_url    VARCHAR(500) NOT NULL                 COMMENT '存储路径',
    file_size   BIGINT       NULL                     COMMENT '文件大小(字节)',
```

## 六、开发规范

### 6.1 分层职责

| 层 | 职责 | 命名规范 |
|---|---|---|
| Controller | 接收请求、参数校验、调用 Service、返回 `Result<T>` | `XxxController`，方法名与业务动作一致 |
| Service | 业务逻辑、事务管理 | 接口 `XxxService` + 实现 `XxxServiceImpl`，继承 `IService<Entity>` |
| Mapper | 数据访问 | `XxxMapper`，继承 `BaseMapper<Entity>`；复杂查询写 XML |
| Entity | 数据库表映射 | `@TableName`、`@TableId`、`@TableLogic`；字段 camelCase 对应表 snake_case |
| DTO | 接收前端参数 | 配合 `@Valid` + `@NotBlank` / `@NotNull` / `@Size` 等校验注解 |
| VO | 返回前端数据 | 隐藏敏感字段(如 password)，按页面需求组装 |

### 6.2 统一响应体

```java
@Data
@AllArgsConstructor
@NoArgsConstructor
public class Result<T> {
    private int code;
    private String message;
    private T data;

    public static <T> Result<T> ok(T data) {
        return new Result<>(200, "success", data);
    }

    public static <T> Result<T> fail(int code, String message) {
        return new Result<>(code, message, null);
    }
}
```

### 6.3 JWT 认证流程

```
1. POST /api/auth/login
   → 验证用户名密码(BCrypt)
   → 生成 JWT(载荷: userId, role, 过期时间)
   → 返回 { token, user }

2. 前端
   → localStorage 存储 token
   → Axios 请求拦截器: headers.Authorization = "Bearer " + token

3. JwtFilter (OncePerRequestFilter)
   → 拦截 /api/** (排除 /api/auth/**)
   → 从 Authorization 头解析 token
   → 验证签名 + 有效期
   → 将用户信息存入 UserContext (ThreadLocal)

4. Controller / AOP
   → 读取 UserContext.getCurrentUser() 进行角色校验
   → 请求结束后 JwtFilter 清理 ThreadLocal
```

### 6.4 权限校验方案

自定义注解 `@RequireRole` + AOP 切面：

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface RequireRole {
    RoleEnum[] value();
}

// Controller 使用
@RequireRole({RoleEnum.TEACHER})
@PostMapping
public Result<Void> createProject(@RequestBody @Valid ProjectDTO dto) { ... }
```

### 6.5 Vue 前端规范

| 项 | 规范 |
|---|---|
| 路由守卫 | `router.beforeEach` 检查 token 和角色，未登录跳转 `/login`，角色不匹配跳转 403 |
| API 层 | 每个模块独立文件 (`api/project.js`)，使用 `request.js` 封装的 Axios 实例 |
| 状态管理 | Pinia 存储 `user` (用户信息) 和 `token` |
| UI 组件 | Element Plus 按需引入 |
| 页面布局 | 顶部导航 + 左侧菜单，菜单根据 `role` 动态渲染 |

### 7.3 数据库连接

1. View → Tool Windows → Database
2. `+` → Data Source → MySQL
3. Host: `localhost`，Port: `3306`，Database: `internship_db`
4. 输入用户名密码 → Download Driver → Test Connection → Apply

### 7.4 前端开发(热重载)

```bash
cd src/main/frontend
npm install
npm run dev     # 启动 Vite (端口 5173)
```

Vite 配置了 `/api` 代理到 `localhost:8080`，浏览器访问 `http://localhost:5173` 即可实现前后端联调。

1. Run → Edit Configurations → `+` → Spring Boot

5. Apply → Run

```bash
mvn clean package -DskipTests
```
