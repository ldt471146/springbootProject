## 背景

管理员用户管理页目前支持审批、启停账号和重置密码，不支持直接删除用户。用户实体 `sys_user` 已具备逻辑删除字段，因此删除功能优先采用逻辑删除，而不是物理删除。

## 目标

- 管理员可删除学生和教师账号
- 删除后账号不再出现在用户列表，也不能继续登录
- 已有关联业务数据的用户不做物理清理，历史记录仍可查看姓名和关联信息

## 方案

### 删除策略

- 采用 MyBatis-Plus 逻辑删除
- 不允许删除当前登录账号
- 不允许删除管理员账号

### 后端

- 新增管理员删除用户接口
- `UserService` 增加删除方法
- 删除前做安全校验
- 为历史展示增加“按 ID 读取已删除用户”的只读查询，避免历史页面姓名变成空值

### 前端

- 在管理员“全部用户”表格中新增“删除”操作
- 删除前弹出确认框
- 删除成功后刷新用户列表和提醒数

## 验证

- 后端编译通过
- 前端构建通过
- 管理员可删除非管理员账号
- 删除后历史记录仍可正常展示用户姓名
